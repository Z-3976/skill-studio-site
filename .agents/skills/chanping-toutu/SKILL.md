# chanping-toutu

## 使用场景

适用于：

- 产品头图
- 团购头图
- 套餐图
- A4 KT板
- A3 KT板
- 健身月卡
- 私教周卡
- 门店宣传图
- 根据 Logo、产品信息、主题色做图

不适用于：

- 普通聊天
- 代码开发
- 非视觉设计任务

## 输入信息要求

优先收集：

- 门店 Logo
- 产品名称
- 价格
- 产品权益 / 产品信息
- 主题色
- 参考样板图（可选）

如果用户明确给了主题色，要把它当成强约束，不要明显跑色。

## 执行流程

### 第一步：检查用户是否提供了关键信息

重点检查：

- Logo
- 产品名称 / 价格 / 权益
- 主题色
- 参考图

其中参考图是可选项，其余信息优先补齐。

### 第二步：如果缺信息，用最少问题补齐

一次性合并追问，不做长篇分析。  
优先补：

1. Logo
2. 产品名、价格、权益
3. 主题色
4. 参考图

### 第三步：整理成统一设计参数

把输入整理成这些参数：

- 设计类型：头图 / 团购头图 / 套餐图 / A4 KT板 / A3 KT板 / 宣传图
- 画面方向：头图默认横版，KT板默认竖版
- 品牌元素：Logo、门店名、品牌识别特征
- 核心卖点：套餐名、价格、权益、适合人群
- 主题色方案：围绕用户主题色展开
- 风格方案：优先参考用户给的样板图

### 第四步：如果环境支持图片生成，直接生成图片

规则：

- 产品头图、团购头图、套餐图默认按 4:3 横版生成
- 中间核心内容控制在 1:1 安全区，避免裁切
- KT板按竖版线下物料逻辑排版
- 生成完成后，如环境支持文件保存，默认保存到桌面

### 第五步：如果环境不支持图片生成，输出最终中文生图提示词

只输出“最终生图提示词”，不输出冗长解释。

## 输出规则

- 若可直接生图：直接生成图片，并简短说明已按 `4:3 + 中间 1:1 安全区` 或 `KT板竖版逻辑` 执行
- 若不可直接生图：只输出“最终生图提示词”
- 不要先长篇分析再做图
- 若环境支持保存文件，成图后默认保存到桌面

## 设计规范

- 风格优先参考用户给的样板图
- 默认尺寸 4:3
- 中间核心内容控制在 1:1 安全区，避免裁切
- 用户主题色作为主色
- 主题色保持大体色差不变，只允许做明暗、渐变和材质延展
- Logo 放上方或视觉核心区
- 主标题大字突出
- 重点突出套餐名、价格、权益
- 中文字体醒目，适合抖音团购、本地生活、健身房宣传
- 少小字
- 整体要高级、清晰、有商业感
- 画面主体必须是健身房相关空间、器械区、镜面墙、前台或真实训练氛围
- 不要出现咖啡厅、家居、办公室、美妆店等无关场景

## 示例

### 示例 1：产品头图

用户输入：

`做一张99元健身月卡头图，Logo用我发的，主题色薄荷青，突出免费淋浴、免费停车、巡场教练和体测。`

期望行为：

- 识别为产品头图
- 按 4:3 横版生成
- 中心 1:1 安全区放 Logo、主标题、价格、权益
- 画面主体必须是健身房器械区和镜面墙

### 示例 2：A3 KT板

用户输入：

`做一张A3 KT板，主标题99健身权益卡，蓝色主题，权益列出来，参考我给的蓝色样板。`

期望行为：

- 识别为 A3 KT板
- 按竖版物料逻辑生成
- 顶部 Logo，中上大标题价格，中部权益清单，下部健身器械氛围

### 示例 3：环境不支持生图

用户输入：

`给我一个99元健身月卡的最终生图提示词，薄荷青主色，高级一点。`

期望行为：

- 不做长解释
- 只输出最终中文生图提示词
